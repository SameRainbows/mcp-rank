# MCP Rank Distribution Drafts

## Main Public Report Copy

MCP discovery is exploding, but trust evidence is still thin.

MCP Rank now has its first real trust layer:

- 18,656+ indexed MCP discovery records
- 50 reviewed or high-risk MCP pages
- 20 Deep Reviews
- 22 Source Reviewed listings
- 0 Maintainer Verified listings

That last number is deliberate.

MCP Rank is not treating directory presence, public metadata, Glama listings, registry listings, or external badges as maintainer confirmation. Maintainer Verified should mean a maintainer actually confirmed source links, install paths, auth behavior, and safety notes.

The first trust-layer report found a pattern:

- Some MCP listings have broken repository links.
- Some are endpoint-only, with little visible provenance.
- Some touch high-impact systems like hosting, DNS, Salesforce admin, analytics uploads, data sandboxes, packet capture, identity/trust scoring, payments, browser sessions, or local files.
- Some are useful enough to review deeply, but still not ready to call "verified."

That distinction matters because MCP installation is becoming less like browsing a plugin directory and more like making a security/product decision. A server can be useful, popular, or listed publicly while still needing clearer source evidence, permission boundaries, retention notes, or maintainer confirmation.

MCP Rank now separates the labels:

- Indexed: discovered from public sources
- Source Reviewed: public links and provenance signals checked
- Deep Review: stronger MCP Rank evidence and risk assessment
- Maintainer Verified: explicit maintainer confirmation

## Reddit / Hacker News

Title: MCP Rank: a trust and provenance index for MCP servers

MCP server directories are useful, but MCP install decisions are starting to look less like directory browsing and more like security/product decisions.

I built MCP Rank to separate public MCP listings by evidence level rather than treating every discovered server as equally trustworthy.

Current state:

- 18,656+ indexed MCP discovery records
- 50 reviewed or high-risk MCP pages
- 20 Deep Reviews
- 22 Source Reviewed listings
- 0 Maintainer Verified listings

The zero is deliberate. MCP Rank does not treat a registry listing, directory listing, public source URL, or external badge as maintainer confirmation.

The first trust-layer report found that MCP provenance is uneven. Some listings had broken repository links. Some were endpoint-only. Some touched high-impact systems such as hosting/DNS, Salesforce admin, analytics uploads, data sandboxes, packet capture, payments, local files, browser sessions, or identity/trust scoring.

The labels are intentionally strict:

- Indexed means discovered from public sources.
- Source Reviewed means public source/provenance links were checked.
- Deep Review means MCP Rank performed a stronger manual evidence and risk assessment.
- Maintainer Verified requires explicit maintainer confirmation.

## Maintainer-Facing Post

If your MCP server appears on MCP Rank, you can now claim or correct the listing.

MCP Rank indexes MCP servers from public sources, but it does not mark listings as Maintainer Verified by default.

The labels are intentionally separate:

- Indexed: discovered from public sources
- Source Reviewed: source/provenance links checked
- Deep Review: stronger MCP Rank evidence and risk assessment
- Maintainer Verified: explicit maintainer confirmation

If a source link is wrong, a repo moved, package metadata is stale, the server description is inaccurate, or the listing misses important auth/safety notes, please send evidence and we will review it.
